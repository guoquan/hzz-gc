## 0x00. Overview (概览)

**HZZ-GC** 是 HZZ 群的专属项目。我们是一个由技术爱好者、工程师、CEO 和产品经理组成的“中年技术中坚”群体。

大家在群里经常发布自己用 AI 开发的、或者随手写的小工具，戏称为“小垃圾”。为了不让这些灵感随着聊天记录流失，我们决定做一个 **GC (Garbage Collection)** 系统，把它们回收、整理、展示出来。

这不是一个为了 KPI 或融资的项目，这是我们的**赛博茶水间**。

## 0x01. Core Concepts (核心概念)

本项目严格采用 **Garbage Collection** 隐喻：

*   **Garbage (垃圾/对象)**:
    *   定义：群友开发的任何小工具、脚本、Demo 或 prompt。
    *   标准：只要能跑一次，或者让作者开心 5 分钟，就是合格的 Garbage。
    *   *System.out.println("各位昨天的小垃圾，请于今日 12 点前投放。");*

*   **Heap (堆/展示区)**:
    *   所有 Garbage 被投放后集中展示的地方。

*   **GC (回收/整理)**:
    *   **Minor GC**: 每日/每周的自动汇总（昨日小垃圾）。
    *   **Major GC**: 周期性的整理、分类、相似度匹配。
    *   **Full GC**: (慎用) 全群围观的年度/季度盘点。

*   **Reachability (可达性)**:
    *   **Reachable**: 有人点赞、有人在用、有人讨论的活跃 Garbage。
    *   **Unreachable**: 长期无人问津的 Garbage（但我们依然保留，纪念逝去的青春）。

*   **Reference Count (引用计数)**:
    *   即“点赞”或“Fork”。引用计数 > 0，说明这个垃圾还有生命力。

## 0x02. Phases (迭代阶段)

### Phase 1: Minor GC (Visualization)
**目标**: 展示。
建立基础的“堆空间”，让大家可以把 Garbage 扔进来。
*   功能：提交 Garbage（标题、作者、描述、链接）、时间轴展示。
*   *Slogan: "Allocating space for your new Garbage..."*

### Phase 2: Major GC (Deduplication & Search)
**目标**: 检索与去重。
发现相似的 Garbage，避免重复造轮子。
*   功能：标签系统、关键词搜索、相似 Garbage 推荐。
*   *Feature: "Duplicate Garbage detected. Reference updated."*

### Phase 3: Semantic GC (AI Integration)
**目标**: 语义化检索。
通过自然语言找到你需要的那个“小垃圾”。
*   场景：“我想找个能批量改文件名的脚本。” -> 自动定位到某位群友 3 个月前发的 Python 脚本。
*   *Prompt: "Find a Garbage that solves X."*

### Phase 4: Reference Counting (Social)
**目标**: 互动。
引入引用计数机制，发现高质量 Garbage。
*   功能：+1 Reference (点赞)、Hot/Cold 分区。
